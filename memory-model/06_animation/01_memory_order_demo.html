seq_cst 顺序一致性
最严格的内存顺序。所有线程看到相同的操作顺序，所有原子操作全局有序。 保证 "既然看到了 A，就一定能看到 A 之前的所有操作"。
// Thread 1 x.store(1, seq_cst); y.store(1, seq_cst); // Thread 2 while (y.load(seq_cst) == 0); assert(x.load(seq_cst) == 1); // 永远成立

acq_rel 获取-释放
用于 read-modify-write 操作（如 exchange）。同时具有 acquire 和 release 的效果： 获取旧值的同时写入新值，并保证同步。
std::atomic<int> turn{0}; // Thread 1: 想获取 turn int old = turn.exchange(1, acq_rel); // old=0 表示获得锁，现在可以安全操作 // Thread 2: 想释放 turn turn.store(0, release);

release / acquire 释放-获取同步
release 写操作与 acquire 读操作形成同步点。release 之后的操作不会被重排到 release 之前； acquire 之前的操作不会被重排到 acquire 之后。这是实现生产者-消费者模式的基础。
std::atomic<bool> ready{false}; int data = 0; // 生产者 (release) data = 42; // 写入数据 ready.store(true, release); // 释放同步 // 消费者 (acquire) while (!ready.load(acquire)); // 获取同步 assert(data == 42); // 一定能看到 data=42

relaxed 松弛顺序
只保证原子性，不保证顺序。编译器/CPU 可以自由重排操作。 适合计数器等不需要同步的场景。
std::atomic<int> x{0}, y{0}; // Thread 1 x.store(1, relaxed); y.store(1, relaxed); // Thread 2 int r1 = y.load(relaxed); int r2 = x.load(relaxed); // r1=0 && r2=0 是可能的！操作被重排了

📊 六种内存顺序对比
内存顺序 用途 可见性 顺序保证 性能
seq_cst 全局同步 全局 强顺序 最慢
acq_rel RMW 操作 配对同步 中顺序 中
release 写操作 配对可见 前序不重排 中
acquire 读操作 配对可见 后续不重排 中
relaxed 计数器 无 无 最快
consume 依赖链 依赖可见 依赖不重排 快
